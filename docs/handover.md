## 1. 시스템 개요

### 목적

Reborn Labs 어드민은 **중고차 딜러 관리 플랫폼**입니다. 차량 재고부터 고객 상담, 판매 등록, 딜러 정산까지 전체 영업 프로세스를 하나의 화면에서 관리합니다.

### 역할 체계

시스템에는 4가지 역할이 있습니다.

| 역할 | 명칭 | 설명 |
|------|------|------|
| `admin` | 경영진 | 전체 기능 접근 가능. 사용자 관리, 감사 로그 포함 |
| `staff` | 직원 | 차량·상담·판매·정산·지출·문서 관리 가능. 사용자 관리·감사 로그 제외 |
| `dealer` | 딜러 | 본인에게 배정된 상담과 본인 판매 내역만 열람 가능 |
| `pending` | 대기 | 계정 생성 직후 상태. 경영진이 역할을 부여하기 전까지 접근 불가 |

### 주요 기능

- 차량 재고 등록 및 상태 관리 (코드 자동 채번)
- 고객 상담 수신 및 딜러 배정
- 판매 등록 (DB 제공 / 자체 판매 구분)
- 딜러별 · 마케팅업체별 정산 확인
- 지출결의 등록 및 증빙 파일 관리
- 공용 문서함 (사업자등록증, 계약서 템플릿 등)
- 전자서명 수집 (모바일 서명 패드)
- 감사 로그 (주요 행위 자동 기록, 경영진만 열람)
- 외부 랜딩페이지 상담 폼 연동 (Google Apps Script)

## 2. 관리자 가이드

> 이 섹션은 비개발자 관리자를 위한 운영 가이드입니다.

### 2.1 로그인 및 초기 설정

1. 브라우저에서 서비스 주소로 접속합니다.
2. 이메일과 비밀번호를 입력하고 로그인합니다.
3. **최초 로그인 시 반드시 비밀번호를 변경해야 합니다.** 변경하지 않으면 일부 기능이 제한됩니다.
4. 로그인 후 왼쪽 사이드바에서 각 메뉴로 이동합니다.

> 주의: 회원가입 기능은 비활성화되어 있습니다. 계정은 경영진이 직접 생성합니다.

---

### 2.2 사용자 관리 (경영진 전용)

**접근 경로:** 사이드바 → 사용자 관리

#### 직원/딜러 계정 초대

1. 사용자 관리 페이지에서 "초대" 버튼을 클릭합니다.
2. 이름, 이메일, 전화번호를 입력합니다.
3. 초대하면 시스템에서 임시 비밀번호가 발급됩니다.
4. 임시 비밀번호를 해당 직원/딜러에게 전달합니다.
5. 초대된 계정은 처음에 `대기(pending)` 상태이므로, 역할을 지정해야 접속이 가능합니다.

#### 역할 변경

1. 사용자 목록에서 해당 계정을 선택합니다.
2. 역할 항목에서 원하는 역할(직원/딜러)을 선택합니다.
3. 저장하면 즉시 반영됩니다. (해당 사용자는 다음 로그인부터 새 역할 적용)

#### 계정 비활성화

1. 사용자 목록에서 해당 계정을 선택합니다.
2. "비활성화" 버튼을 클릭합니다.
3. 비활성화된 계정은 로그인 시 접근 차단 화면으로 이동합니다.

---

### 2.3 차량 관리

**접근 경로:** 사이드바 → 차량 관리 (경영진/직원)
**접근 경로:** 사이드바 → 차량 목록 (딜러 — 매입가·마진 숨김)

#### 차량 등록

1. 차량 관리 페이지에서 "차량 등록" 버튼을 클릭합니다.
2. 다음 항목을 입력합니다:
   - 제조사 (예: 현대, 기아)
   - 모델명 (예: 소나타, K5)
   - 연식
   - 주행거리 (km)
   - **매입가** (원, 딜러에게 비공개)
   - 판매가 (원)
   - 보증금, 월납입료 (해당 시)
   - 차량 사진 (복수 첨부 가능)
3. 등록하면 **차량 코드(RB-연도-번호)**가 자동으로 부여됩니다. (예: RB-2026-001)

#### 차량 상태

차량 상태는 시스템이 자동으로 변경합니다.

| 상태 | 의미 | 전환 조건 |
|------|------|----------|
| 출고가능 | 판매 가능한 재고 | 초기 등록 시 |
| 상담중 | 해당 차량으로 상담 진행 중 | 상담에 차량 연결 시 자동 |
| 판매완료 | 판매 확정 | 판매 등록 시 자동 |

#### 차량 삭제

- 차량은 **완전 삭제가 아닌 숨김 처리**됩니다. (이력 보존을 위함)
- 단, 진행 중인 상담이 있는 차량은 삭제할 수 없습니다.

> 주의: 딜러는 차량의 매입가와 마진을 볼 수 없습니다.

---

### 2.4 상담 관리

**접근 경로:** 사이드바 → 상담 관리 (경영진/직원) / 내 상담 (딜러)

#### 상담 접수 방법

- **랜딩페이지 폼**: 외부 고객이 랜딩페이지에서 문의하면 자동으로 상담 목록에 추가됩니다.
- **직접 등록**: 상담 관리 페이지에서 "상담 등록" 버튼으로 직접 입력할 수 있습니다.

#### 중복 감지

- 동일한 전화번호로 재문의가 들어오면 **중복 문의**로 표시됩니다.
- 중복 상담은 목록에서 별도로 확인할 수 있습니다.

#### 딜러 배정 및 마케팅 업체 기록

1. 상담 목록에서 해당 상담을 클릭합니다.
2. 담당 딜러를 선택하고, 해당 상담을 연결한 마케팅 업체가 있다면 기록합니다.
3. 배정된 딜러는 로그인 후 본인 상담 목록에서 해당 상담을 확인할 수 있습니다.

#### 상담 기록 타임라인

- 딜러 또는 직원이 상담 진행 내용을 메모로 남길 수 있습니다.
- 상담 상세 페이지에서 타임라인 형태로 모든 기록을 확인할 수 있습니다.

#### 상담 상태 흐름

```
신규 → 상담중 → 차량대기 → 판매완료
                         → 거부
```

| 상태 | 설명 |
|------|------|
| 신규 | 접수된 상담 |
| 상담중 | 딜러가 상담을 진행 중 |
| 차량대기 | 구매 의사 확인, 차량 준비 대기 |
| 판매완료 | 판매 등록 시 자동 전환 |
| 거부 | 상담 거절 처리 |

### 2.5 판매 등록

**접근 경로:** 사이드바 → 판매 관리

#### DB 제공 판매 (마케팅 업체 연계)

> 마케팅 업체가 제공한 고객 상담 데이터로 성사된 판매

1. 상담 상세 페이지로 이동합니다.
2. "판매 완료" 버튼을 클릭합니다.
3. 판매 차량을 선택합니다.
4. 등록 시 수당과 수수료가 자동 계산됩니다:
   - 딜러 수당: **50만원**
   - 마케팅 수수료: **70만원**

#### 자체 판매 (딜러 자체 발굴)

> 딜러가 직접 고객을 발굴하여 성사된 판매

1. 판매 관리 페이지에서 "직접 판매 등록" 버튼을 클릭합니다.
2. 판매 차량, 담당 딜러를 선택합니다.
3. 등록 시 수당과 수수료가 자동 계산됩니다:
   - 딜러 수당: **100만원**
   - 마케팅 수수료: 없음

#### 전자서명 수집

- 판매 등록 화면에서 고객의 전자서명을 모바일로 받을 수 있습니다.
- 서명은 한 번만 저장되며 이후 수정이 불가합니다.

#### 계약서 업로드

- 판매 상세 페이지에서 계약서 파일을 첨부할 수 있습니다.
- 파일 크기 제한: 10MB

#### 판매 취소

- 경영진 또는 직원만 판매를 취소할 수 있습니다.
- 취소 시 취소 사유를 반드시 입력해야 합니다.
- 취소 후 차량 상태는 자동으로 복원됩니다.

---

### 2.6 정산 확인 (경영진/직원)

**접근 경로:** 사이드바 → 정산

#### 딜러별 정산

1. 조회 기간을 선택합니다.
2. 딜러를 선택하면 해당 기간의 판매 건수와 수당 합계를 확인할 수 있습니다.

#### 마케팅 업체별 정산

1. 조회 기간을 선택합니다.
2. 마케팅 업체명으로 검색하면 DB 제공 건수와 수수료 합계를 확인할 수 있습니다.

#### 월별 요약

- 정산 페이지 상단에 이번 달 전체 판매 수, 딜러 수당 합계, 마케팅 수수료 합계가 표시됩니다.

---

### 2.7 지출결의 (경영진/직원)

**접근 경로:** 사이드바 → 지출결의

#### 지출 등록

1. "지출 등록" 버튼을 클릭합니다.
2. 다음 항목을 입력합니다:
   - 지출 일자
   - 금액 (원)
   - 지출 목적
   - 영수증 이미지 (복수 첨부 가능, 각 10MB 이하)
3. 저장하면 목록에 추가됩니다.

#### 증빙 미리보기

- 지출 목록에서 항목을 클릭하면 첨부된 영수증 이미지를 미리 볼 수 있습니다.

---

### 2.8 문서함

**접근 경로:** 사이드바 → 문서함

문서는 3가지 카테고리로 분류됩니다:
- **사업자등록증**: 회사 사업자 관련 서류
- **계약서 템플릿**: 표준 계약서 양식
- **기타**: 위 분류에 해당하지 않는 문서

#### 업로드 (경영진/직원)

1. 카테고리를 선택합니다.
2. 파일을 첨부합니다. (10MB 이하)
3. 업로드 버튼을 클릭합니다.

#### 다운로드

- 모든 역할(경영진/직원/딜러)이 문서를 다운로드할 수 있습니다.

#### 삭제 (경영진 전용)

- 경영진만 문서를 삭제할 수 있습니다.

---

### 2.9 감사 로그 (경영진 전용)

**접근 경로:** 사이드바 → 감사 로그

시스템에서 중요한 행위가 발생할 때 자동으로 기록됩니다. 경영진만 열람 가능합니다.

자동 기록 항목:
- 사용자 초대
- 역할 변경
- 차량 삭제
- 판매 완료
- 판매 취소
- 랜딩페이지 상담 접수 (Google Apps Script 경유)

## 3. 기술 정보

> 이 섹션은 개발자 및 시스템 관리자를 위한 기술 문서입니다.

### 3.1 기술 스택

| 항목 | 기술 |
|------|------|
| 프레임워크 | Next.js 16 (App Router) |
| 언어 | TypeScript |
| 데이터베이스 | Supabase (PostgreSQL) |
| 인증 | Supabase Auth |
| 파일 스토리지 | Supabase Storage |
| UI 라이브러리 | Tailwind CSS v4 + shadcn/ui (Base UI 기반) |
| 배포 | Vercel |
| 외부 연동 | Google Apps Script (랜딩 폼 → 상담 DB) |

---

### 3.2 DB 구조

#### 테이블 목록 (10개)

| 테이블 | 용도 |
|--------|------|
| `profiles` | 사용자 프로필 및 역할 관리 (auth.users 확장) |
| `vehicles` | 차량 재고 (매입가·판매가·마진·상태) |
| `consultations` | 고객 상담 수신 및 딜러 배정 |
| `consultation_logs` | 상담 진행 타임라인 메모 |
| `sales` | 판매 완료 기록 (수당·수수료 포함) |
| `delivery_checklists` | 차량 인도 체크리스트 (딜러용) |
| `expenses` | 지출결의 기록 |
| `documents` | 공용 문서 보관 |
| `audit_logs` | 시스템 감사 로그 (불변, INSERT 전용) |
| `rate_limits` | API 레이트 리미팅 요청 추적 |

#### RPC 함수 (4개)

| 함수 | 동작 |
|------|------|
| `insert_consultation_from_gas` | Google Apps Script에서 상담 폼 제출 수신 → consultations 삽입 + 감사 로그 |
| `complete_sale` | 판매 완료 처리: 행 잠금 → 수당 계산 → sales 삽입 → 차량/상담 상태 갱신 → 감사 로그 |
| `cancel_sale` | 판매 취소: 상담/차량 상태 복원 → 취소 사유 기록 → 감사 로그 |
| `get_dashboard_stats` | 역할별 대시보드 통계 반환 (admin/staff: 전체 현황, dealer: 개인 현황) |

#### 주요 트리거 (8개)

| 트리거 | 동작 |
|--------|------|
| `trg_generate_vehicle_code` | 차량 INSERT 시 RB-연도-순번 형식 코드 자동 채번 (연도별 리셋, 최대 999대) |
| `custom_access_token_hook` | JWT 발급 시 profiles.role을 JWT claims에 삽입 (user_role claim) |
| 상태 전이 트리거 (consultations) | 상담 상태 변경 시 차량 상태 자동 동기화 |
| updated_at 자동 갱신 트리거 | 각 테이블 UPDATE 시 updated_at 현재 시각으로 갱신 |

### 3.3 환경변수

Vercel Dashboard 또는 `.env.local` 파일에 다음 변수를 설정해야 합니다.

| 변수명 | 용도 | 필수 여부 |
|--------|------|----------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 프로젝트 URL | 필수 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase 익명 키 (클라이언트용) | 필수 |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase 서비스 롤 키 (서버 전용, 절대 노출 금지) | 필수 |
| `GAS_WEBHOOK_URL` | Google Apps Script 웹훅 URL (랜딩 폼 연동) | 선택 |

> 주의: `SUPABASE_SERVICE_ROLE_KEY`는 RLS를 우회하는 전체 권한 키입니다. 절대 클라이언트(브라우저)에 노출되어서는 안 됩니다. `NEXT_PUBLIC_` 접두사 없이 서버 전용으로만 사용합니다.

### 3.5 배포 절차

#### 일반 코드 배포

```bash
git push origin main
```

Vercel에 GitHub 레포지토리가 연결되어 있어 `main` 브랜치에 push하면 자동으로 빌드 및 배포됩니다.

2. 프로젝트 선택 → Settings → Environment Variables
3. 변수 추가 또는 수정
4. 재배포 필요 시 Deployments 탭 → 최신 배포 → Redeploy

```bash
# Supabase CLI 사용 (로컬에서 실행)
supabase db push
```

또는 Supabase Dashboard → SQL Editor에서 마이그레이션 파일 내용을 직접 실행합니다.

### 3.6 Supabase 대시보드 설정

#### JWT Custom Claims Hook 활성화 (필수)

이 설정이 없으면 역할 기반 접근 제어가 정상 동작하지 않습니다.

1. [Supabase Dashboard](https://supabase.com/dashboard) 접속
2. 프로젝트 선택 → Authentication → Hooks
3. "Customize Access Token (JWT) Claims" 섹션 찾기
4. Hook 타입: **PostgreSQL Function** 선택
5. 함수: **`public.custom_access_token_hook`** 선택
6. 저장

> 이 설정 전까지는 미들웨어(`proxy.ts`)가 service_role 키로 profiles 테이블을 직접 조회하여 동작합니다. 성능 최적화를 위해 반드시 활성화를 권장합니다.

#### 회원가입 비활성화 확인

외부 회원가입을 차단하려면 다음을 확인합니다:

1. Supabase Dashboard → Authentication → Providers
2. Email 항목의 **"Enable Sign Up"** 비활성화
3. 소셜 로그인 항목 전체 비활성화

#### Storage 버킷 확인

6개 버킷이 생성되어 있는지 확인합니다:

| 버킷명 | 용도 | 폴더 구조 |
|--------|------|----------|
| `vehicles` | 차량 사진 | `{vehicle_id}/{filename}` |
| `checklists` | 인도 체크리스트 파일 | `{dealer_id}/{filename}` |
| `contracts` | 계약서 파일 | `{sale_id}/{filename}` |
| `signatures` | 전자서명 | `{sale_id}/{filename}` |
| `receipts` | 영수증 이미지 | `{expense_id}/{filename}` |
| `documents` | 공용 문서 | `{category}/{filename}` |

Supabase Dashboard → Storage에서 6개 버킷이 존재하는지 확인합니다.

#### pg_cron 설정 (선택)

`rate_limits` 테이블은 오래된 레코드를 주기적으로 정리해야 합니다.

Supabase Dashboard → Database → Extensions에서 `pg_cron` 활성화 후:

```sql
SELECT cron.schedule(
  'cleanup-rate-limits',
  '0 * * * *',
  $$DELETE FROM rate_limits WHERE requested_at < now() - interval '1 hour'$$
);
```

## 4. 트러블슈팅

### 로그인이 안 됩니다

**원인 1:** 이메일/비밀번호 오류
- 올바른 이메일과 비밀번호를 입력했는지 확인합니다.
- 경영진에게 비밀번호 초기화를 요청합니다.

**원인 2:** profiles 레코드 없음 (개발자)
- Supabase Dashboard → Table Editor → `profiles` 테이블에서 해당 사용자 UUID로 레코드가 존재하는지 확인합니다.
- auth.users에는 있지만 profiles에 없는 경우 레코드를 수동으로 삽입합니다.

---

### `/unauthorized` 페이지로 이동됩니다

**원인 1:** 역할이 `pending` 상태
- 경영진이 사용자 관리에서 역할(직원/딜러)을 부여해야 합니다.

**원인 2:** 계정 비활성화
- Supabase Dashboard 또는 사용자 관리 페이지에서 해당 계정의 `is_active` 값을 확인합니다.

**원인 3:** profiles 레코드 없음
- profiles 테이블에 해당 user id의 레코드가 없을 때 발생합니다.

---

### 딜러에게 상담이 보이지 않습니다

- 상담 관리에서 해당 상담의 "담당 딜러" 항목에 딜러가 배정되어 있는지 확인합니다.
- Supabase Dashboard에서 `consultations` 테이블의 `assigned_dealer_id` 컬럼 값을 확인합니다.

---

### 이미지/파일 업로드에 실패합니다

**원인 1:** 파일 크기 초과
- 모든 버킷의 최대 파일 크기는 10MB입니다.

**원인 2:** Storage 버킷 정책 오류 (개발자)
- Supabase Dashboard → Storage → Policies에서 해당 버킷에 INSERT 정책이 활성화되어 있는지 확인합니다.
- `007_storage.sql` 마이그레이션이 정상 적용되었는지 확인합니다.

---

### 판매 등록에 실패합니다

**원인 1:** 차량 상태 확인
- 이미 판매된 차량(`sold`)이나 삭제된 차량(`deleted`)은 판매 등록이 불가합니다.
- 차량 관리에서 차량 상태를 확인합니다.

**원인 2:** 상담 상태 확인
- 상담 상태가 `신규` 또는 `상담중` 단계에 있어야 판매 등록이 가능합니다.

---

### 랜딩페이지 상담 폼이 DB에 저장되지 않습니다

- Google Apps Script 웹훅 연동을 확인합니다.
- `GAS_WEBHOOK_URL` 환경변수가 올바르게 설정되어 있는지 확인합니다.
- 자세한 설정은 `docs/gas-integration-guide.md`를 참고합니다.
